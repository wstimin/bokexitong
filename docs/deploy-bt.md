# 宝塔网页向导部署

宝塔这一版也只走网页，不需要在服务器里敲命令。你只要上传项目，然后在宝塔里点几下就能完成部署、反向代理和 HTTPS。

## 先准备

- 服务器已经安装宝塔面板。
- 服务器已经安装 Docker 或 Docker 管理器。
- 宝塔已经安装 Nginx。
- 域名已经解析到服务器 IP。
- 已经放行 `80` 和 `443` 端口。

## 第一步: 上传项目

在宝塔的文件管理里上传项目压缩包，解压到你想放的位置，比如：

```text
/www/wwwroot/bokexitong
```

## 第二步: 创建应用

在宝塔页面中依次点击：

1. 「Docker」
2. 「Compose」或「编排」
3. 「添加项目」
4. 选择项目目录
5. 选择 `docker-compose.yml`
6. 选择 `.env`
7. 点击保存并启动

## 第三步: 填写配置

打开 `.env`，把这几项填好：

```env
MYSQL_ROOT_PASSWORD=你的强密码
BLOG_JWT_SECRET=至少32位随机字符串
BLOG_ADMIN_INITIAL_PASSWORD=至少8位后台初始密码
BLOG_DOMAIN=你的域名
FRONTEND_HTTP_BIND=127.0.0.1:18080
```

`FRONTEND_HTTP_BIND=127.0.0.1:18080` 的意思是：前端只在本机监听，后面交给宝塔的网站反向代理。

## 第四步: 创建网站

在宝塔页面中依次点击：

1. 「网站」
2. 「添加站点」
3. 域名填你的域名
4. PHP 版本选「纯静态」
5. 保存
6. 进入站点设置
7. 打开「反向代理」
8. 目标地址填：

```text
http://127.0.0.1:18080
```

9. 保存并启用

## 第五步: 申请 HTTPS

在站点设置里继续点击：

1. 「SSL」
2. 选择 Let's Encrypt
3. 勾选你的域名
4. 申请
5. 开启「强制 HTTPS」

## 登录信息

后台用户名固定是：

```text
admin
```

后台初始密码在 `.env` 里：

```text
BLOG_ADMIN_INITIAL_PASSWORD=...
```

## 你会看到什么

- 宝塔负责应用编排
- 宝塔负责域名
- 宝塔负责证书
- 项目前端只监听本机 `127.0.0.1:18080`

## 注意

- 不要把后端 `8080` 直接暴露到公网。
- 不要把 MySQL `3306` 直接暴露到公网。
- 网站反向代理地址就是 `http://127.0.0.1:18080`。
