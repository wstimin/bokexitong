# 宝塔网页部署

宝塔这一版也只走网页，不需要在服务器里敲命令，也不需要手动编辑环境变量。上传 `bokexitong-linux.tar.gz` 构建包后，在宝塔里启动项目、创建反向代理网站，然后通过浏览器打开安装向导即可。

这里不需要宝塔应用商店里的运行环境插件；但项目本身包含 MySQL、后端和前端三个容器，所以仍然需要用宝塔 Docker 的 Compose / 编排功能把 `docker-compose.yml` 启动起来。

这个包已经包含后端 `app.jar`、前端 `dist` 和启动所需文件。首次打开站点时会自动进入 `/install` 安装向导，在页面里填写数据库连接、站点名称、域名和管理员密码；信息正确后系统会自动建表、写入配置并重启，随后进入正常站点。

## 前提

- 服务器已经安装宝塔面板。
- 服务器已经安装 Docker，且宝塔可以管理容器或编排。
- 宝塔已经安装 Nginx。
- 域名已经解析到服务器 IP。
- 已经放行 `80` 和 `443` 端口。

## 第一步: 上传并解压

在宝塔的文件管理里上传 `bokexitong-linux.tar.gz`，解压到你想放的位置，比如：

```text
/www/wwwroot/bokexitong
```

解压后你会看到这些文件：

```text
docker-compose.yml
.env
backend/app.jar
backend/Dockerfile.runtime
frontend/dist/
frontend/Dockerfile.runtime
sql/personal_blog.sql
```

## 第二步: 启动容器

在宝塔页面中依次点击：

1. 「Docker」
2. 「Compose」或「编排」
3. 「添加项目」
4. 选择项目目录
5. 选择 `docker-compose.yml`
6. 环境变量文件选择 `.env`
7. 点击保存并启动

构建包里的 `.env` 默认是：

```text
FRONTEND_HTTP_BIND=18080
```

容器启动后先用下面的地址确认安装向导能打开：

```text
http://127.0.0.1:18080/install
```

这个地址是在服务器本机侧验证前端入口用的。面板创建网站后，用户实际访问你的域名即可。

## 第三步: 创建网站

在宝塔页面中依次点击：

1. 「网站」
2. 「添加站点」
3. 域名填你的域名
4. PHP 版本选「纯静态」
5. 保存
6. 进入站点设置
7. 打开「反向代理」
8. 目标地址填：

```text
http://127.0.0.1:18080
```

如果访问域名出现 502，再把目标地址改成服务器真实 IP 对应的 `http://服务器IP:18080` 或宿主机网关地址。

然后保存网站配置，后续申请 HTTPS 并通过域名访问即可。

9. 保存并启用

## 第四步: 申请 HTTPS

在站点设置里继续点击：

1. 「SSL」
2. 选择 Let's Encrypt
3. 勾选你的域名
4. 申请
5. 开启「强制 HTTPS」

## 第五步: 打开安装页

容器启动后，访问你的域名：

```text
https://你的域名/
```

如果还没配置 HTTPS，可以先访问：

```text
http://你的域名/
```

首次访问会自动进入 `/install` 安装向导。页面里填写：

- 数据库主机：使用内置 MySQL 时填 `mysql`。
- 数据库端口：默认 `3306`。
- 数据库名：默认 `personal_blog`。
- 数据库用户：使用内置 MySQL 时填 `root`。
- 数据库密码：使用内置 MySQL 时可以留空，系统会自动使用包内 `.env` 的密码；使用外部数据库时填写真实密码。
- 站点名称和访问域名。
- 管理员密码：至少 8 位，安装后用于登录后台。

点击「开始安装」后，系统会自动连接数据库、创建缺失表、保存配置并重启后端。页面提示正在重启时等几秒，再访问域名即可进入站点首页。

## 登录信息

后台用户名固定是：

```text
admin
```

后台密码就是安装向导里填写的管理员密码。

## 注意

- 不要把后端 `8080` 直接暴露到公网。
- 不要把 MySQL `3306` 直接暴露到公网。
- 网站反向代理地址先使用 `http://127.0.0.1:18080`；如果面板反向代理访问这个地址出现 502，再改为 `http://服务器IP:18080` 或宿主机网关地址。

## 502 排查

如果访问 `/install` 就显示 502，按顺序检查：

1. 先在服务器本机侧确认 `http://127.0.0.1:18080/install` 能打开。能打开说明容器正常，是宝塔网站反向代理没有连到这个入口。
2. 宝塔网站反向代理地址先填写 `http://127.0.0.1:18080`；如果面板反向代理无法访问宿主机回环地址，再改成 `http://服务器IP:18080` 或宿主机网关地址。
3. 在 Docker / Compose 页面确认 `frontend`、`backend`、`mysql` 三个容器都是运行中。
4. 如果 `127.0.0.1:18080/install` 也打不开，查看 `frontend` 容器日志；如果页面能打开但提交安装时报错，再检查安装页填写的数据库信息或查看 `backend` 容器日志。
